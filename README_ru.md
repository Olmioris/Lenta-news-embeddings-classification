# Классификация тематик новостей Lenta.ru (100k документов).

Этот проект реализует полный, воспроизводимый NLP‑пайплайн для классификации русскоязычных новостных статей по темам, используя классические методы построения эмбеддингов. В него входят загрузка данных, предобработка, токенизация, векторизация с использованием нескольких стратегий эмбеддингов, обучение моделей, оценка и сравнение качества эмбеддингов.

## 1. Обзор проекта.

Репозиторий содержит end‑to‑end пайплайн классификации текстов на основе подвыборки из 100 000 документов корпуса новостей Lenta.ru.
Цель — оценить, насколько далеко можно продвинуться, используя классические методы эмбеддингов (Word2Vec, Navec, RusVectōrēs, TF‑IDF‑взвешенные эмбеддинги) на большом реальном русскоязычном датасете.

Пайплайн включает:

 - эффективную предобработку русскоязычного текста;
 - стратифицированное разбиение (60/20/20);
 - обучение собственных эмбеддингов Word2Vec;
 - загрузку предобученных Navec и RusVectōrēs;
 - построение sentence embeddings (усреднение);
 - TF‑IDF‑взвешивание эмбеддингов;
 - обучение Logistic Regression;
 - intrinsic‑оценку эмбеддингов;
 - финальную оценку на отложенной тестовой выборке.

## 2. Датасет.

- источник: Lenta.ru (Corus / Kaggle);

- размер: 100 000 документов;

- поля: title, text, topic;

- удалены редкие классы (<2 объектов) для корректной стратификации.

Каждая запись представляет собой одну новостную статью с тремя полями:

 - title — заголовок,
 - text — полный текст,
 - topic — тема.

## 3. Пайплайн предобработки.

Предобработка включает:

- приведение к нижнему регистру;
- удаление неалфавитных символов;
- нормализацию пробелов;
- простую токенизацию (для Word2Vec / Navec);
- морфологическую токенизацию (lemma_POS) для RusVectōrēs;
- удаление пустых текстов;
- фильтрацию классов с <2 объектов.

Зачем морфологическая токенизация?

RusVectōrēs обучены на токенах формата lemma_POS (например, москва_PROPN, идти_VERB). Использование такого же формата значительно улучшает качество эмбеддингов.

## 4. Разбиение на выборки.

- train: 60%

- validation: 20%

- test: 20%

- стратификация по темам

- random_state = 42 для воспроизводимости

## 5. Используемые модели эмбеддингов.

В проекте сравниваются четыре стратегии:

- Word2Vec (обученный на корпусе Lenta.ru)

- Navec (предобученные эмбеддинги Natasha)

- RusVectōrēs (лемматизированные эмбеддинги с POS‑тегами)

- TF‑IDF‑взвешенные эмбеддинги (почти на уровне Navec)

Результаты на валидации (macro F1):

- Navec — 0.630 (лучший результат)
- TF‑IDF‑взвешенные эмбеддинги — 0.625
- Word2Vec — 0.614
- RusVectōrēs (с морфологией) — 0.566

Intrinsic‑оценка включает:

- поиск ближайших соседей;
- odd‑one‑out;
- проверки семантической близости.

Примеры сохранены в:

reports/intrinsic_examples.txt

### 5.1 Обоснование гиперпараметров Word2Vec

**vector_size = 100**  
Оптимальный баланс для корпуса среднего размера. Большие размерности требуют больше данных и дают минимальный прирост качества.

**window = 5**  
Хорошо работает для русского языка: улавливает смысловые связи, не размывая контекст.

**min_count = 5**  
Удаляет редкие слова‑шум, но сохраняет важные сущности (имена, города, организации).

**workers = 4**  
Ускоряет обучение без лишней нагрузки на систему.

**sg = 1 (skip‑gram)**  
Лучше работает с редкими словами, которых много в новостях (фамилии, организации).

**epochs = 10**  
Оптимум для корпуса среднего размера: меньше — недообучение, больше — минимальный прирост.

**seed = RANDOM_STATE**  
Гарантирует воспроизводимость результатов.

## 9. Анализ ошибок.

Типичные ошибки классификации:

- World → Russia

- Russia → World

- Economy → Russia

- Lifestyle → World

Причина — пересекающаяся лексика и похожие контекстные признаки между темами, что является известным ограничением классических моделей bag‑of‑embeddings.

Сильные классы:
- Россия
- Мир
- Культура
- Спорт
- Экономика

Слабые классы:
- Библиотека (1 пример)
- Наука и техника (54 примера)

Macro F1 = 0.63 отражает сложность редких классов.

Главное ограничение качества — сильный дисбаланс классов, а не сами эмбеддинги.
